# Novelist — кликабельный прототип маркетплейса для инди-авторов книг

## О проекте

**Novelist** — это социальная сеть / маркетплейс, где инди-авторы публикуют книги, иллюстраторы добавляют визуал, рецензенты оставляют ревью, а читатели покупают и читают контент.

Этот репозиторий содержит полностью кликабельный HTML-прототип, построенный по принципу **Code-First**: вместо дизайн-макетов сразу вёрстка с интеграцией реальных плагинов, переходами между страницами и рабочими интерактивными элементами.

### Причина code fist прототипирования

Прототип создан в **2019 году** как док-во актуальности и преимущества code fist подхода в UI/UX дизайне. Спустя 7 лет, в **2026-м**, с появлением AI-ассистентов (вроде Claude, GPT, Copilot) Code-First подход доказал своё превосходство: генерация кода по текстовому описанию позволяет создавать прототипы на порядок быстрее, чем отрисовка макетов, а правки вносятся естественным языком без переключения между инструментами.

Прототип создавался для **проверки гипотез вживую** — на реальных устройствах, в реальных браузерах, на реальных экранах разного размера. Только так можно было:

- Увидеть, как плагины (Select2, Summernote, Owl Carousel, Chart.js) работают на мобильных и десктопе
- Потрогать скролл, ховеры, фокусы, адаптивность
- Показать живой продукт людям и **получить отклик по факту**, а не по картинке из макета
- Итеративно вносить правки без цикла «дизайнер → разработчик → дизайнер»

## Технологический стек

- **Bootstrap 4.4.1** — сетка, компоненты, утилиты
- **jQuery 3.4.1** — DOM-манипуляции, события
- **Font Awesome 5.11.2** — иконки
- **Owl Carousel 2** — карусели (новинки, популярное, новые авторы)
- **Select2** — множественный выбор с поиском (жанры, теги, циклы)
- **Summernote** — WYSIWYG-редактор
- **Chart.js** — графики статистики книг
- **Starrr** — звёздный рейтинг
- **bs-custom-file-input** — кастомные файловые поля

### Кастомные файлы

- `css/modification.css` — утилитарные классы, оверрайды Bootstrap, галерея, тултипы
- `css/reader.css` — стили читалки
- `css/media-player.css` — стили медиа-плеера
- `js/forallpages.js` — общие скрипты (тултипы, галерея, кнопка «наверх», toggle deck/list, очистка полей)
- `js/reader.js` — скрипты читалки (цветовые схемы, размер шрифта, букмаркинг, скрытие хедера при скролле)
- `js/guestbook.js` — гостевые книги (комментарии, ветки)
- `js/selectizze.js` — кастомный селект (запасной)

## Пользовательские сценарии (user flows)

| Сценарий | Путь |
|----------|------|
| Гость → регистрация | `index-guest.html` → модалка Sign Up → `registration-success.html` |
| Гость → вход | модалка Sign In → `index-guest.html` → хедер меняется на авторизованный |
| Поиск книг → чтение | `books.html` → `book.html` → `reader.html` |
| Чтение книги с закладками | `reader.html` — клик по правому краю абзаца → букмарк |
| Чтение с заметками | Выделить текст → меню: Sticky Note / Quotation / Found Error |
| Просмотр профиля автора | `profile.html` → книги → рецензии → активность |
| Workroom автора | `my-books.html` → создание/редактирование книги |
| Управление циклами | `my-cycles.html` → `my-cycle.html` (настройки, обложка, привязка книг) |
| Кошелёк | `wallet.html` → транзакции → пополнение/вывод |
| Настройки | `account.html` → профиль → security → удаление |
| Каталог авторов | `writers.html` → фильтры → профиль |

## Code-First vs Design-First: преимущества подхода

Традиционный **Design-First** подход выглядит так: средство макетирования (Figma подобные программы) → утверждение макетов → передача разработчикам → вёрстка → интеграция → обнаружение неучтённых состояний → переделка макетов → снова вёрстка.

**Code-First** (как сделан этот прототип) переворачивает процесс: прототип сразу создаётся в коде с использованием реального стека (Bootstrap, jQuery, плагины). Вот ключевые преимущества:

### 1. Интерактивность, а не картинки
В макетах нельзя «пощупать» скролл, ховеры, фокусы, анимации, переключение табов, выпадающие меню, модальные окна. В этом прототипе всё работает как в продакшене — можно проверить юзабилити на реальных взаимодействиях.

### 2. Реальные плагины
Select2, Summernote, Chart.js, Owl Carousel — это не заглушки. Сразу видно, как они ведут себя на мобильных, как выглядят тултипы, как работают древовидные комментарии. В Design-First это выясняется только на этапе разработки, когда макеты уже утверждены.

### 3. Адаптивность «из коробки»
Bootstrap-сетка сразу даёт представление о том, как интерфейс выглядит на всех разрешениях. В средствах макетирования резиновые макеты — отдельная боль, их редко прорабатывают для всех брейкпоинтов. Здесь достаточно растянуть браузер.

### 4. Переходы между страницами
Можно пройти полный пользовательский путь (гость → каталог → книга → читалка → профиль → настройки) и убедиться, что ни одна ссылка не ведёт в никуда, а состояние хедера/сайдбара меняется корректно. В средстве макетирования это имитируется в лучшем случае прототипом с экранами-картинками.

### 5. Сокращение цикла «дизайн → разработка»
Дизайнер/продуктолог, владеющий Bootstrap, может сделать прототип, который разработчику останется только «оживить» данными. Никакой передачи макетов, нарезки, споров о пикселях — всё уже в коде.

### 6. Раннее обнаружение проблем
Состояния ошибок (неверный пароль, удаление аккаунта, пустые списки), длинные строки (обрезание текста), переполнение контентом — всё это видно сразу, а не обнаруживается через месяц после утверждения примитивных макетов.

### 7. Единый источник правды
Нет рассинхрона между макетами и кодом. Изменение вносится один раз — прямо в HTML. Никаких «а у нас в макете кнопка синяя, а в коде зелёная».

### 8. AI-нативность
Code-First — единственный подход, который масштабируется AI-ассистентами. 2026 год показал: LLM генерируют и редактируют код на порядок эффективнее, чем макеты. Прототип в коде — это промпт-дружелюбный артефакт, который можно развивать диалогом, а не перерисовывать вручную.
